## 1. Purpose

Establish steps to issue, modify, suspend, and revoke badges and keys/cards for GridSite facilities and sites.

## 2. Roles

Facilities (owner), Guard Desk, Site Manager, Cardholder, Sponsor/Escort, ISMS.

## 3. Prerequisites

Access portal enrollment; identity verification; background check completion for vendor/facility roles; defined
access profile; badge stock; kiosk and guard terminal operational.

## 4. Procedure

1. Request & Approval: Sponsor submits access request with justification and access profile; Facilities approves per
   least‑privilege and SoD.
2. Enrollment: Cardholder enrolls at kiosk (badge, face, PIN) with ID verification; guard terminal can enroll if kiosk
   unavailable.
3. Issuance: Activate exterior‑to‑lobby access only until kiosk sign‑in; interior access is enabled upon sign‑in and
   automatically revoked at sign‑out.
4. Changes: Modify access upon role changes; re‑verify identity for sensitive areas; log all admin actions.
5. Lost/Forgotten: Deactivate lost badges immediately; issue temporary credential after face/PIN and ID check; order
   replacement if lost.
6. Revocation: Upon offboarding or suspension, deactivate credentials; collect badge; document completion.
7. Evidence: Export access logs and admin actions to evidence store; if forwarding isn’t available, perform periodic
   manual exports.

## 5. Records & Evidence

Access requests/approvals, enrollment records, badge inventory, access/admin logs, revocation confirmations.

## 6. Metrics

Time‑to‑issue, revocation SLA, lost badge rate, audit sample pass rate.
