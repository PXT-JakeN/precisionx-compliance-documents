## 1. Purpose

Define provisioning, modification, and deprovisioning of access to financial systems with SoD and MFA.

## 2. Roles

Requester, Manager Approver, Finance Systems Owner, ISMS/GRC, Internal Audit.

## 3. Procedure

1. Provisioning: Require access request form with justification and role; enforce SSO via Microsoft 365 SSO/SAML
   where available; enforce MFA; map to least‑privilege roles and SoD constraints.
2. Changes: Process role changes via documented approval; log all admin actions; review elevated access monthly.
3. Reviews: Perform quarterly user access reviews; remediate exceptions; maintain evidence exports if systems cannot
   forward.
4. Deprovisioning: Remove access within 24 hours of offboarding or role removal; confirm in evidence register.

## 4. Records & Evidence

Access requests/approvals, role mappings, review results, admin logs, deprovision confirmations.

## 5. Metrics

Provisioning SLA compliance, review completion rate, access exceptions count, time‑to‑revoke.
