Conventions
- Use YAML metadata at the top of each file (see templates).
- Maintain standard sections: Purpose, Scope, Roles and Responsibilities, Policy Statements, Exceptions,
  Compliance Measurement, Enforcement, Revision History.
- Link to supporting documents with relative paths (e.g., ../standards/Access-Control-Standard.md).
- Update version and effective_date upon substantive changes; maintain additive Revision History table.
